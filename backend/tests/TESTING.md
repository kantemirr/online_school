# Тестирование CodeKids (ВКР, пп. 3.7–3.8)

Документ описывает функциональные тест-кейсы по ролям и механикам (**Табл. 18**),
сквозной контрольный пример (**Табл. 19**) и порядок запуска. Вывод `pytest -v`
со всеми зелёными тестами — материал для **Рис. 26**.

## Стек и запуск

- **Бэкенд:** `pytest` + `pytest-asyncio`. Юнит-тесты — чистая логика без инфраструктуры;
  интеграционные — против реального Postgres/Redis через ASGI (`httpx.ASGITransport`),
  запускаются внутри контейнера.
- **Фронтенд:** `vitest` — юнит-тесты чистых утилит.

```bash
# Бэкенд (все тесты, подробно — для Рис. 26)
docker compose exec backend pytest -v

# Бэкенд (кратко)
docker compose exec backend pytest -q          # 60 passed

# Живой прогон песочницы требует поднятого worker (есть в dev-стеке);
# иначе test_code_sandbox_live пропускается (skip), суит остаётся зелёным.

# Фронтенд
docker compose exec frontend npm test          # 9 passed
```

## Табл. 18 — Функциональные тест-кейсы

### Юнит-тесты (чистая логика, без инфраструктуры) — `backend/tests/`

| № | Область | Что проверяется | Ожидаемый результат | Файл |
|---|---------|------------------|---------------------|------|
| U1 | Безопасность | Хеш argon2 (round-trip), JWT access/refresh, истечение, неверный тип токена | Пароль верифицируется; токены декодируются; просроченный/чужой тип отвергнут | `test_security.py` |
| U2 | Модель данных | Конфигурация мапперов 22 сущностей | Все мапперы валидны | `test_models.py` |
| U3 | Прохождение | Разблокировка уроков, пересчёт прогресса (урок/модуль/курс) | Первый открыт, остальные закрыты; % считается верно | `test_learning_logic.py` |
| U4 | Грейдинг | Оценка квиза (single/multiple/matching), агрегация вердикта кода, эвристика подсказок | Зачёт при ≥60%; вердикт passed/failed/partial; совет по stderr | `test_grading.py` |
| U5 | Геймификация | `is_earned` по типам условий, `compute_streak` (сегодня/вчера/разрыв) | Достижение выдаётся при выполнении; серия растёт/сбрасывается | `test_gamification.py` |
| U6 | Расписание | Валидация времени занятия (конец > начала) | Некорректный интервал отвергнут | `test_scheduling.py` |
| U7 | Оплата | Расчёт периода/суммы по плану, `has_access` (free/период/курс/истёкший) | Цены и доступ корректны | `test_payments.py` |
| U8 | Аналитика | Доля с guard деления на ноль (`safe_pct`) | 0 при нулевом знаменателе | `test_analytics.py` |
| U9 | Админ | `protected_change_reason` (self / last_admin / смена роли) | Блокируется самоблок и потеря последнего админа | `test_admin.py` |
| U10 | Smoke | `/health` | `{status:ok, db:true, redis:true}` | `test_smoke.py` |

| № | Роль / механика | Сценарий | Ожидаемый результат | Файл |

## Табл. 19 — Контрольный пример (сквозной сценарий)

Реализован одним тестом `test_control_example.py`:

| Шаг | Действие | Проверка |
|-----|----------|----------|
| 1 | Родитель регистрируется, подтверждает email, входит | 201; verify 204; токен выдан |
| 2 | Родитель заводит профиль ребёнка (ник/логин/PIN) | 201, есть `user_id` |
| 3 | Родитель оформляет абонемент: checkout → «оплата» (имитация) | `paid`, есть `receipt_no`, абонемент `active`, квитанция |
| 4 | Ребёнок входит по логину и PIN | Токен ученика выдан |
| 5 | Ребёнок записывается на демо-курс | 201 |
| 6 | Ребёнок проходит уроки и решает квиз | `passed`, урок закрыт |
| 7 | Начисляются XP и достижение | `xp>0`, `first_lesson` earned |
| 8 | Ученику приходит уведомление | список непуст, `unread>0` |
| 9 | Родитель видит прогресс ребёнка | ребёнок в обзоре, детальный отчёт |
| 10 | Родитель открывает печатный HTML-отчёт | 200, в HTML есть имя ребёнка (выходной документ) |
