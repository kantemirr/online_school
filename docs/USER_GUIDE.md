# Руководство пользователя CodeKids (Приложение В)

Краткое руководство по работе с платформой для четырёх ролей. Скриншоты экранов
вставляются в текст ВКР (Приложение В) — места для них помечены `[скриншот: …]`.

## Запуск и доступ

```bash
docker compose up --build          # поднять стек
docker compose exec backend python -m app.db.reset_and_seed   # чистая БД с демо-данными
```

Приложение — <http://localhost:8080>, Swagger API — <http://localhost:8080/docs>,
письма (MailHog) — <http://localhost:8025>.

## Ученик (8–14 лет)

Вход: `/login/child` — крупный экран, логин + PIN на цифровой клавиатуре.

1. **Главная** (`/home`) — приветствие маскота Кодика, плитки XP/серия/ранг/награды, «Продолжить обучение». `[скриншот: дашборд ученика]`
2. **Каталог** (`/catalog`) — поиск и фильтры по направлению/уровню; карточки курсов.
3. **Курс → запись** — на бесплатный курс запись сразу; платный требует абонемента родителя.
4. **Карта прохождения** (`/learn/:courseId`) — уроки открываются последовательно (замок на закрытых).
5. **Урок** — теория (Markdown) + видео + задания.
6. **Задания:**
   - **Квиз** — выбор ответа(ов), мгновенная оценка, зачёт при ≥ 60 %.
   - **Код** — редактор CodeMirror, «Запустить проверку» → автопроверка в песочнице → вердикт по тестам, при провале — ИИ-подсказка (без готового решения). `[скриншот: песочница с вердиктом]`
   - **Проект** — загрузка файла/ссылки, проверяет преподаватель.
7. **Награды** (`/achievements`) и **Рейтинг** (`/leaderboard`) — достижения и место в топе.
8. **Расписание** (`/schedule`) и **колокольчик уведомлений** в сайдбаре.

## Родитель

Вход: `/login`. Навигация: Главная / Дети / Оплата.

1. **Главная** (`/parent`) — плитки расходов семьи, карточки детей с прогрессом. `[скриншот: кабинет родителя]`
2. **Дети** (`/parent/children`) — добавить ребёнка (ник, дата рождения, логин, PIN), редактировать, удалить.
3. **Отчёт ребёнка** (`/parent/children/:id`) — прогресс по курсам, посещаемость, достижения; «Открыть отчёт» → печатный HTML (выходной документ). `[скриншот: отчёт об успеваемости]`
4. **Оплата** (`/parent/payments`) — оформить абонемент (Месяц 990 ₽ / Год 9900 ₽ / Курс), «оплатить (демо)» → квитанция; история, отмена.

## Преподаватель

Вход: `/login`. Навигация: Группы / Проверка.

1. **Группы** (`/teacher`) — создать группу по курсу.
2. **Группа** (`/teacher/groups/:id`) — состав (поиск и добавление учеников), занятия (дата + ссылка на видеоконференцию), **посещаемость** (present/absent/excused). `[скриншот: посещаемость группы]`
3. **Аналитика группы** — средний прогресс/посещаемость, по каждому ученику.
4. **Очередь проверки** (`/teacher/queue`) — проекты на ручную проверку; балл + отзыв → урок закрывается, ученику уходит уведомление. `[скриншот: проверка работы]`

## Администратор

Вход: `/login`. Навигация: Сводка / Пользователи / Контент / Платежи / Группы / Журнал.

1. **Сводка** (`/admin`) — метрики платформы (пользователи, активность, популярные курсы, выручка). `[скриншот: админ-сводка]`
2. **Пользователи** (`/admin/users`) — поиск/фильтры, блокировка, смена роли персонала, сброс пароля, создание преподавателя/админа.
3. **Контент** (`/admin/content`) — CRUD курсов/модулей/уроков/заданий, публикация; конструктор квизов и тест-кейсов кода. `[скриншот: редактор курса]`
4. **Реестры** — все платежи и группы.
5. **Журнал** (`/admin/audit`) — аудит ключевых действий администраторов.

## Сквозной сценарий (контрольный пример)

1. Родитель `parent@codekids.ru` оформляет абонемент → «оплачивает» → получает квитанцию.
2. Ребёнок `kid` / `1234` входит, записывается на курс, проходит урок и квиз, отправляет код в песочницу → `passed` → получает XP и достижение.
3. Родитель видит обновлённый отчёт ребёнка и открывает печатную версию.
4. Преподаватель отмечает посещаемость и проверяет проект.
5. Администратор видит платёж в реестре и метрики в сводке.

Полная матрица тест-кейсов и контрольный пример — в [`backend/tests/TESTING.md`](../backend/tests/TESTING.md).
